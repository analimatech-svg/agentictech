# Pipeline: Análise de Codebase

> Auditoria técnica de uma base de código existente: qualidade, dívida técnica,
> segurança, observabilidade e aderência às boas práticas.
> Indicado para Consultores de IA em qualquer nível Maestro.

## Quando usar

- O cliente já tem um sistema em produção e quer entender o estado real
- O time assumiu um legado e precisa mapear antes de mexer
- Existe suspeita de problema de performance, segurança ou arquitetura

## Nível mínimo na Escala Maestro

Aprendiz (com assistência do Judge em cada etapa)

---

## Sequência de skills

| Ordem | Skill | Output principal |
|---|---|---|
| 1 | `skills/quality/code-review` | Relatório de qualidade de código |
| 2 | `skills/quality/security` | Relatório de vulnerabilidades |
| 3 | `skills/quality/best-practices` | Relatório de aderência às boas práticas |
| 4 | `skills/docs/doc-architecture` | Documentação da arquitetura atual (reversa) |
| 5 | `skills/docs/doc-adr` | ADR das decisões históricas identificadas |
| 6 | `skills/docs/doc-bug-report` | Registro de problemas encontrados |
| 7 | `skills/docs/doc-status-report` | Relatório executivo do estado do sistema |

---

## Critério de conclusão

- Os três relatórios de qualidade estão gerados e revisados
- A arquitetura atual está documentada
- O relatório executivo está pronto para apresentação ao cliente ou time

## Nota sobre privacidade

Nenhum código ou dado do cliente é enviado para provedores externos sem consentimento explícito.
O Consultor de IA é responsável por configurar a chave de API de forma segura e
por garantir que o contexto fornecido às skills não inclui informações confidenciais além do necessário.
